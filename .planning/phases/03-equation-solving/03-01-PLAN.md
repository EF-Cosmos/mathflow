    **GREEN Phase -- Implement solve_service.py:**
    Create `code/mathflow-new/backend/app/services/solve_service.py`:

    1. Import: `sympy`, `Symbol`, `Eq`, `latex`, `parse_latex`, `solve`, `reduce_inequalities`, `linsolve`, `Poly`, `Rational`, `simplify`, `StrictLessThan`, `LessThan`, `StrictGreaterThan`, `GreaterThan`, `And`, `Or` from sympy. Import `parse_latex_safe` and `normalize_latex` from `sympy_service` module (relative import: `from .sympy_service import parse_latex_safe`).

    2. Helper function `_find_variable(expr)` -- scan the SymPy expression for free symbols, return the first one that is a single letter (exclude Greek symbols like Delta). If multiple variables found, return the first alphabetically. This is used by equation and inequality solvers.

    3. `solve_equation_with_steps(latex_str: str) -> dict`:
       - Parse with `parse_latex_safe(latex_str)` -- returns Eq object for equations.
       - Call `_find_variable` to get the variable.
       - Check if the expression is polynomial in the variable: try `Poly(expr.lhs - expr.rhs, var)`.
       - If degree 1 (linear): extract a = poly.coeff_monomial(var), c = poly.coeff_monomial(1). Generate steps: (1) "原方程" with original LaTeX, (2) if c != 0, "两边减 {c}" with the equation after subtracting c from both sides, (3) "两边除以 {a}" with the solved equation. Use `latex(Eq(var, solution))` for the final step.
       - If degree 2 (quadratic): extract a, b, c. Generate steps: (1) "原方程", (2) "识别系数: a = {a}, b = {b}, c = {c}", (3) "计算判别式: \\Delta = {b}^2 - 4 \\times {a} \\times {c} = {delta_value}" -- compute delta = b**2 - 4*a*c, render the discriminant LaTeX, (4) "应用求根公式" with the quadratic formula LaTeX. Use `solve(eq, var)` to get the actual solutions. Render each solution with `latex(Eq(var, sol))`.
       - If degree > 2 or not polynomial: use `solve(eq, var)` directly, wrap result in minimal steps (original equation + solution).
       - For fractional equations (expression contains fractions with variable in denominator): use `solve(eq, var)` to find solutions, then check each solution by substituting back into the original equation's denominators. If a solution makes any denominator zero, filter it out and add a step: "x = {value} 使分母为零，舍去". Per research Open Question 2 recommendation.
       - Return `{"result": final_latex, "steps": steps_list, "verified": False}`.

    4. `solve_inequality_with_steps(latex_str: str) -> dict`:
       - Parse with `parse_latex_safe(latex_str)`. Returns relational objects (StrictLessThan, LessThan, etc.).
       - Determine inequality direction from the parsed object type.
       - Call `reduce_inequalities([inequality_expr], var)` where inequality_expr is constructed based on the relational type.
       - Handle all result types per research Pitfall 3:
         - `True` -> "解为全体实数", intervals is empty list, result_text = "x \\in \\mathbb{R}"
         - `False` -> "该不等式无解", intervals is empty list, result_text = "\\text{无解}"
         - `Eq(var, value)` -> single point, intervals with lower=upper=value, both strict=False
         - `And(...)` -> extract interval boundaries by iterating args, checking each for relational type
         - `Or(...)` -> recursively extract intervals from each And clause
       - Build `intervals` list for frontend NumberLine component: `[{lower: float|null, upper: float|null, lower_strict: bool, upper_strict: bool}]`.
       - Generate steps: (1) "原不等式" with original LaTeX, (2) "化简" with rearranged form, (3) "解集" with the solution LaTeX.
       - Return `{"result": result_latex, "steps": steps_list, "intervals": intervals_data, "verified": False}`.

    5. `solve_system_with_steps(equations: list[str], variables: list[str]) -> dict`:
       - Parse each equation with `parse_latex_safe`. Each returns an Eq object.
       - Validate: number of equations >= number of variables, all equations parse successfully.
       - Extract expressions: `eq.lhs - eq.rhs` for each equation.
       - Create Symbol objects from variable names.
       - Call `linsolve(exprs, vars)` to get solution set.
       - Handle: EmptySet -> no solution, FiniteSet with tuple -> extract variable assignments.
       - Generate steps: (1) "方程 {i+1}" for each equation, (2) "求解结果" with final variable assignments like "x = 2, y = 1".
       - Return `{"result": final_latex, "steps": steps_list, "verified": False}`.

    ```python
    # ==================== 求解模型 ====================

    class SolveStep(BaseModel):
        description: str = Field(..., description="变换说明，如'两边减 3'")
        latex: str = Field(..., description="中间结果 LaTeX")

    class SolveEquationRequest(BaseModel):
        latex: str = Field(..., description="方程的 LaTeX 表达式，如 2x + 3 = 7")

    class SolveEquationResponse(BaseModel):
        result: str = Field(..., description="最终解的 LaTeX")
        steps: List[SolveStep] = Field(..., description="求解步骤列表")
        verified: bool = Field(default=False, description="结果是否经过验证")

    class SolveInequalityRequest(BaseModel):
        latex: str = Field(..., description="不等式的 LaTeX 表达式，如 2x + 3 > 7")

    class IntervalData(BaseModel):
        lower: Optional[float] = Field(None, description="区间下界，null 表示负无穷")
        upper: Optional[float] = Field(None, description="区间上界，null 表示正无穷")
        lower_strict: bool = Field(True, description="下界是否为开区间（不包含）")
        upper_strict: bool = Field(True, description="上界是否为开区间（不包含）")

    class SolveInequalityResponse(BaseModel):
        result: str = Field(..., description="解集的 LaTeX 表达式")
        steps: List[SolveStep] = Field(..., description="求解步骤列表")
        intervals: List[IntervalData] = Field(default_factory=list, description="解集区间数据，供前端数轴图使用")
        verified: bool = Field(default=False, description="结果是否经过验证")

    class SolveSystemRequest(BaseModel):
        equations: List[str] = Field(..., description="方程列表，每个元素是一个方程的 LaTeX")
        variables: List[str] = Field(..., description="变量名列表，如 ['x', 'y']")

    class SolveSystemResponse(BaseModel):
        result: str = Field(..., description="最终解的 LaTeX")
        steps: List[SolveStep] = Field(..., description="求解步骤列表")
        verified: bool = Field(default=False, description="结果是否经过验证")
    ```
